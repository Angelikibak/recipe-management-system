Βάση Δεδομένων – Οδηγίες Αναδημιουργίας

Η εφαρμογή χρησιμοποιεί SQLite (τοπική βάση ως αρχείο) και όχι MySQL.
Η SQLite δεν απαιτεί εγκατάσταση server, ούτε δημιουργία user/ρόλου/password.

1) Πού βρίσκεται η βάση
Η βάση αποθηκεύεται ως αρχείο:
- recipes.db

Στο project που παραδίδεται περιλαμβάνεται ήδη ένα αρχείο recipes.db.
Αν διαγραφεί, δημιουργείται ξανά αυτόματα με την πρώτη εκτέλεση της εφαρμογής.

2) Πώς δημιουργούνται οι πίνακες (schema)
Το σχήμα της βάσης ορίζεται στο αρχείο:
- src/main/resources/db/schema.sql

Κατά την εκκίνηση της εφαρμογής εκτελείται αυτόματα η αρχικοποίηση της βάσης
από την κλάση:
- recipes.database.Database

Συγκεκριμένα:
- DB URL: jdbc:sqlite:recipes.db
- Μέθοδος: Database.initialize()
Η initialize() φορτώνει το db/schema.sql και εκτελεί τα SQL statements.
Επιπλέον, γίνονται μικρές migrations (ALTER TABLE) για να προστεθούν columns
αν δεν υπάρχουν ήδη.

3) Πώς τρέχει η εφαρμογή
Απαιτείται JDK 17 (όπως ορίζεται στο pom.xml).

Σε IntelliJ:
- Άνοιγμα του project
- Ρύθμιση Project SDK: Java 17
- Run της κλάσης:
  recipes.Main

Κατά το Run εκτελείται πρώτα:
- Database.initialize()
και μετά ανοίγει το Swing UI.

4) Σημείωση για χρήστες/permissions
Δεν υπάρχουν χρήστες/ρόλοι στη SQLite όπως στη MySQL.
Η εφαρμογή γράφει/διαβάζει απευθείας από το τοπικό αρχείο recipes.db.
